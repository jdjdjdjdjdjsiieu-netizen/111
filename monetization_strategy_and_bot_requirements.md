# Анализ монетизационной стратегии и требования к ИИ-боту

## 1. Оптимальная стратегия заработка
На основе маркетинг-плана "Свой в Альфе" (баллы, квалификация, бонусы) определена следующая стратегия для максимизации дохода при минимальном участии владельца бота:

| Стратегический фокус | Целевое действие бота | Обоснование (Максимизация дохода) |
| :--- | :--- | :--- |
| **Привлечение клиентов (1-я линия)** | Генерация трафика и автоматизированный онбординг. | Основа дохода, источник личных баллов (40+ обязательно). |
| **Рекомендация баллоемких продуктов** | ИИ-анализ потребностей пользователя ТГ и персонализированная рекомендация. | Максимизация баллов с каждого клиента. Фокус на "привязывающих" продуктах (Дебетовая/Кредитная карта, Кредиты). |
| **Стимулирование целевых действий** | Автоматизированные напоминания и инструкции для активации продукта (траты, пополнения). | Получение максимального количества баллов (до 19 баллов за карту + действия). |
| **Рекрутинг партнеров (2-я/3-я линии)** | Автоматизированная воронка рекрутинга для пользователей, проявивших интерес к заработку. | Построение структуры для получения баллов со 2-го и 3-го поколений, а также бонусов за квалификацию (Бонусный дуэт, Бонус за все поколения). |
| **Повышение квалификации** | Сочетание личных продаж и рекрутинга для достижения уровня А8+. | Открытие доступа к самым крупным бонусам (до 5 млн руб.) и максимальной стоимости балла. |

## 2. Требования к функциям ИИ в Telegram-боте

Для реализации стратегии бот должен обладать следующими ИИ-функциями:

| Функция ИИ | Описание | Техническая реализация (Предварительно) |
| :--- | :--- | :--- |
| **Анализ потребностей клиента (Core)** | На основе диалога с пользователем (вопросы о финансовом положении, целях, возрасте, наличии продуктов Альфа-Банка) определить наиболее подходящий и баллоемкий продукт. | LLM (например, `gpt-4.1-mini` через OpenAI API) для классификации и генерации персонализированного ответа. |
| **Генерация персонализированного контента** | Создание уникальных, привлекательных текстов для рекомендации продукта и призыва к целевому действию. | LLM для генерации маркетинговых текстов (копирайтинг). |
| **Автоматизированный онбординг партнера** | Предоставление пошаговых инструкций, ответов на вопросы по маркетинг-плану и мотивация для новичков-партнеров. | LLM + база знаний (FAQ по "Свой в Альфе") для интерактивного обучения. |
| **Оптимизация воронки продаж** | Анализ поведения пользователей в боте (отказы, задержки) и автоматическая корректировка последовательности сообщений (A/B-тестирование). | Простая логика на основе статистики + LLM для генерации вариантов сообщений. |

## 3. Архитектурные требования (Современная архитектура)

Для обеспечения минимального участия, масштабируемости и надежности, архитектура должна быть:

1.  **Микросервисной/Модульной:** Разделение логики бота (Telegram API), бизнес-логики (расчет баллов, воронки) и ИИ-сервиса.
2.  **Асинхронной:** Использование фреймворка, поддерживающего асинхронность (например, `aiogram` или `telebot` с асинхронными библиотеками) для обработки большого количества пользователей.
3.  **СУБД:** Использование PostgreSQL или SQLite (для MVP) для хранения данных пользователей, их статусов, истории рекомендаций и статистики.
4.  **ИИ-интеграция:** Использование внешнего API (OpenAI) для ИИ-функций, что позволяет легко обновлять модель без изменения основной логики бота.

**Вывод:** Ключевой элемент — это ИИ-модуль, который заменит ручной анализ потребностей клиента и подбор продукта, а также автоматизирует рекрутинг и обучение партнеров. Это соответствует цели "минимальное участие".
